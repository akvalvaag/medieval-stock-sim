# Feature Expansion Design ŌĆö Medieval Market Simulator
**Date:** 2026-03-23
**Status:** Approved

## Overview

Add 8 new features to the medieval stock market simulator: Seasons, NPC Bots, Limit Orders, Moneylender, Warehousing Costs, Price Alerts, Achievements, and Net Worth Chart.

Architecture: **Service Pipeline**. Each new concern is a focused `@Component` with a `processTick()` method. `MarketEngine` orchestrates them in sequence each tick.

---

## Backend

### Tick Order (MarketEngine)
1. `seasonEngine.advanceTick()` ŌĆö roll season, get modifiers
2. Reprice all goods (existing logic + season nudge)
3. `botService.processTick()` ŌĆö bots trade
4. `limitOrderService.processAll(portfolios)` ŌĆö trigger pending orders
5. `moneylenderService.processAll(portfolios)` ŌĆö compound interest + default seizure
6. `warehousingService.processAll(portfolios)` ŌĆö deduct holding fees
7. Broadcast `MarketState` (now includes `season`, `seasonTicksRemaining`, per-player `limitOrders`)

### SeasonEngine

**Responsibility:** Track a 4-season cycle; expose passive per-tick drift nudges.

- Each season = 60 ticks (~5 min at 5s/tick)
- Cycle: SPRING ŌåÆ SUMMER ŌåÆ AUTUMN ŌåÆ WINTER ŌåÆ repeat
- `advanceTick()` increments counter; rolls season at 60
- `getModifiers()` returns `Map<String, Double>` of per-tick % nudge for affected goods

**Season modifiers (applied as additive drift each tick):**

| Season | Boosts | Suppresses |
|--------|--------|------------|
| ­¤ī▒ Spring | Grain +8%, Wool +8%, Livestock +8%, Fish +5% | Coal ŌłÆ10%, Candles ŌłÆ10% |
| ŌśĆ’ĖÅ Summer | Spices +10%, Herbs +12%, Ale +5%, Cloth +8% | Grain ŌłÆ10% |
| ­¤Źé Autumn | Grain +8%, Timber +10%, Ale +10%, Coal +8%, Leather +5% | ŌĆö |
| ŌØä’ĖÅ Winter | Coal +20%, Candles +15%, Salt +10%, Spices +8%, Grain +12% | Fish ŌłÆ10%, Livestock ŌłÆ8% |

Season modifiers are scaled down to a small per-tick nudge (e.g. divide by 100) so they accumulate gradually rather than shocking prices.

**MarketState additions:** `season` (string), `seasonTicksRemaining` (int).

### BotService

**Responsibility:** Simulate 5 NPC traders who affect prices and appear on the scoreboard.

- Bots created at startup; registered in `SessionRegistry` with names prefixed `[bot]`
- Starting gold: 300g; bot portfolios are full `Portfolio` objects
- Bot names generated by `NameGenerator` with `[bot] ` prefix
- Every 3 ticks, each bot picks a random good and decides:
  - `currentPrice < 80% of basePrice` AND has gold ŌåÆ buy 1ŌĆō5 units
  - `currentPrice > 120% of basePrice` AND holds some ŌåÆ sell 1ŌĆō5 units
  - Otherwise ŌåÆ 50/50 buy or sell 1ŌĆō3 units (if affordable / has holdings)
- Bots use `TradeService` (fees apply, supply pressure applies)
- Bots appear on scoreboard; no WebSocket connection needed

### LimitOrderService

**Responsibility:** Execute deferred buy/sell orders when price conditions are met.

**Model: `LimitOrder`**
```
id          UUID
goodName    String
direction   BUY | SELL
quantity    int
targetPrice double
```

- BUY triggers when `currentPrice Ōēż targetPrice`
- SELL triggers when `currentPrice Ōēź targetPrice`
- Max 3 pending orders per portfolio
- Orders stored as `List<LimitOrder>` in `Portfolio`
- `processAll()` iterates every portfolio's orders each tick; triggered orders execute via `TradeService`
- If execution throws (insufficient funds/holdings) ŌåÆ order is cancelled silently
- Triggered or cancelled orders are removed and included in the next market state broadcast to the owning session

**New endpoints:**
- `POST /api/orders` ŌĆö `{goodName, direction, quantity, targetPrice}` ŌåÆ `OrderResponse`
- `DELETE /api/orders/{id}` ŌĆö cancels a pending order

### MoneylenderService

**Responsibility:** Allow players to borrow gold at compounding interest; seize assets on default.

- Loan tracked in `Portfolio`: `double loanAmount`
- Borrow limit: `min(2 ├Ś currentGold, 5000g)`; minimum borrow: 50g
- Interest: **2% per tick** (compounding)
- Each tick:
  1. `loanAmount *= 1.02`
  2. Check solvency: if `loanAmount > gold + holdingsValue` ŌåÆ seize
  3. Seizure: sort holdings by `currentPrice ├Ś qty` ascending, sell cheapest first via `TradeService` until debt covered; any remaining debt forgiven
- Repay: pay `min(gold, loanAmount)` from player's gold; clears loan if fully repaid

**New endpoints:**
- `POST /api/borrow` ŌĆö `{amount}` ŌåÆ `LoanResponse{loanAmount, gold}`
- `POST /api/repay` ŌåÆ `LoanResponse{loanAmount, gold}`

**Portfolio additions:** `double loanAmount` (default 0.0), getters/setters.

---

### WarehousingService

**Responsibility:** Charge a small per-tick fee on held inventory to discourage passive hoarding.

- Fee rate: **0.5% of held inventory value** per tick
- Merchant class: **0.25%** (half rate)
- `holdingsValue = ╬Ż (qty ├Ś currentPrice)` for all held goods
- Deduction skipped if gold would go below 0
- Silent ŌĆö no notification to client

## Frontend

### Season Indicator (Header)
- Display in header bar alongside tick counter
- Shows icon + name (e.g. `ŌØä’ĖÅ Winter`) and ticks remaining
- Small horizontal progress bar showing season progress
- Data source: `season` and `seasonTicksRemaining` from `MarketState`

### Net Worth Chart (Portfolio Panel)
- Small bar sparkline below the net worth figure
- Tracked client-side in Vue: append current net worth each tick (keep last 20 values)
- Same visual style as market sparklines; green when trending up, red when down
- No backend change needed

### Limit Orders Panel (Portfolio Panel)
- Shows pending orders as a list (e.g. `SELL Iron Ōēź 65g ├Ś 5`)
- Cancel button (Ō£Ģ) per order
- "Add Order" button opens an inline form: good selector, BUY/SELL toggle, quantity input, target price input
- Slot counter: `x/3 slots`
- Orders synced from server on each `MarketState` message (server echoes player's orders back)

### Moneylender Panel (Portfolio Panel)
- Shows current debt in red (if any): `Debt: 250g (+2%/tick)`
- "Repay" button (disabled if no debt or no gold)
- "Borrow" button opens amount input with max displayed
- Hidden/collapsed when debt = 0 and player hasn't interacted

### Price Alerts (Market Table)

- Each tick: compare old vs new price; if threshold crossed ŌåÆ flash the row + show a brief banner (same style as event banner, but amber)
- Active alert shown as gold bell (­¤öö amber); inactive as dim grey

## Data Flow Changes

- `MarketState` (WebSocket broadcast): add `season`, `seasonTicksRemaining`, `loanAmount` (per-session), `limitOrders` (per-session list)
- `SessionStartResponse`: add `loanAmount`, `limitOrders`
- `TradeResponse`: add `loanAmount` (to reflect seizure during reprice)
- New response DTOs: `OrderResponse`, `LoanResponse`

---

## Out of Scope

- Achievements persisted server-side (localStorage only per user decision)
- Price alert persistence across page refresh (session-only)
- Bot difficulty settings
- Seasonal events (separate from the random event engine)
